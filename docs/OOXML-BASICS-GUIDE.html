üåê What is OOXML?

Office Open XML (OOXML) is the file format used by Microsoft Word, Excel, and PowerPoint since Office 2007. It was standardized as ECMA-376 and later as ISO/IEC 29500.

The key insight is simple: a .docx file is just a ZIP archive containing XML files.

üí° Try It Yourself

Rename any .docx file to .zip, then extract it. You'll see the XML files inside!

Why This Matters

Understanding OOXML means you can:

  • Programmatically create Word documents without Microsoft Office
  • Fix corrupted documents by editing the XML directly
  • Build automation tools that manipulate documents precisely
  • Understand why Word behaves weirdly (spoiler: it's the XML)

üî¢ Numbering Basics

OOXML numbering has a two-layer architecture:

Abstract Definition
(w:abstractNum)
The "template"
‚¨áÔ∏è
Instance
(w:num)
A usage of the template
‚¨áÔ∏è
Paragraph
(w:numPr)
References the instance

Why Two Layers?

This separation allows:

  • Reuse: Multiple lists can share the same format definition
  • Overrides: An instance can modify the abstract definition without changing others
  • Flexibility: You can have 10 instances of "1, 2, 3..." format, each counting independently
‚ÑπÔ∏è Key Insight

A document can have at most TWO numbering.xml files ‚Äî one for the main document and one for the glossary. This constraint is why exhibits and schedules often conflict with main document numbering.

üìê Abstract Numbering Definitions

The <w:abstractNum> element defines the "template" for a numbering scheme. Here's a complete example:

<w:abstractNum w:abstractNumId="0">
    <!-- Unique identifier for this abstract definition -->
    <w:nsid w:val="099A081C"/>
    
    <!-- Type: singleLevel, multilevel, or hybridMultilevel -->
    <w:multiLevelType w:val="multilevel"/>
    
    <!-- Level 0: Roman numerals (I, II, III) -->
    <w:lvl w:ilvl="0">
        <w:start w:val="1"/>
        <w:numFmt w:val="upperRoman"/>
        <w:lvlText w:val="%1."/>
        <w:lvlJc w:val="left"/>
        <w:pPr>
            <w:ind w:left="720" w:hanging="360"/>
        </w:pPr>
    </w:lvl>
    
    <!-- Level 1: Capital letters (A, B, C) -->
    <w:lvl w:ilvl="1">
        <w:start w:val="1"/>
        <w:numFmt w:val="upperLetter"/>
        <w:lvlText w:val="%2."/>
        <w:lvlJc w:val="left"/>
        <w:pPr>
            <w:ind w:left="1440" w:hanging="360"/>
        </w:pPr>
    </w:lvl>
    
    <!-- Level 2: Arabic numerals (1, 2, 3) -->
    <w:lvl w:ilvl="2">
        <w:start w:val="1"/>
        <w:numFmt w:val="decimal"/>
        <w:lvlText w:val="%3."/>
        <w:lvlJc w:val="left"/>
    </w:lvl>
</w:abstractNum>

Key Elements Explained

Element Purpose Example Values
w:start Starting number for this level 1, 5, 100
w:numFmt Number format type decimal, upperRoman, lowerLetter, bullet
w:lvlText Display template (%1 = level 1 number) %1., (%1), Article %1
w:lvlJc Number justification left, center, right
w:pPr Paragraph properties (indentation) Contains w:ind for left/hanging indent
w:pStyle Links this level to a paragraph style Heading1, ListParagraph

All numFmt Values

decimal          ‚Üí 1, 2, 3, 4, 5...
upperRoman       ‚Üí I, II, III, IV, V...
lowerRoman       ‚Üí i, ii, iii, iv, v...
upperLetter      ‚Üí A, B, C, D, E...
lowerLetter      ‚Üí a, b, c, d, e...
ordinal          ‚Üí 1st, 2nd, 3rd...
cardinalText     ‚Üí one, two, three...
ordinalText      ‚Üí first, second, third...
bullet           ‚Üí ‚Ä¢ (or custom symbol)
none             ‚Üí (no visible number)

üéØ Numbering Instances

The <w:num> element creates an instance of an abstract definition. Paragraphs reference instances, not abstract definitions directly.

<w:num w:numId="1">
    <!-- Points to abstractNumId="0" -->
    <w:abstractNumId w:val="0"/>
</w:num>

<w:num w:numId="2">
    <!-- Same abstract, but with an override -->
    <w:abstractNumId w:val="0"/>
    <w:lvlOverride w:ilvl="0">
        <w:startOverride w:val="5"/>
    </w:lvlOverride>
</w:num>

Level Overrides

The <w:lvlOverride> element lets you modify specific levels without creating a new abstract definition:

  • startOverride: Start numbering at a different number
  • lvl: Completely replace the level definition
üí° Pro Tip

When you "Restart Numbering" in Word, it creates a new <w:num> instance pointing to the same abstract definition, with a <w:startOverride> setting the start back to 1.

üîó Paragraph References

In document.xml, paragraphs reference numbering via <w:numPr> inside their properties:

<w:p>
    <w:pPr>
        <!-- This paragraph uses numbering -->
        <w:numPr>
            <w:ilvl w:val="0"/>  <!-- Level 0 (first level) -->
            <w:numId w:val="1"/> <!-- References w:num with numId="1" -->
        </w:numPr>
    </w:pPr>
    <w:r>
        <w:t>This is Article I</w:t>
    </w:r>
</w:p>

<w:p>
    <w:pPr>
        <w:numPr>
            <w:ilvl w:val="1"/>  <!-- Level 1 (second level) -->
            <w:numId w:val="1"/> <!-- Same numbering instance -->
        </w:numPr>
    </w:pPr>
    <w:r>
        <w:t>This is Section A</w:t>
    </w:r>
</w:p>

The Two Ways to Apply Numbering

1. Direct Reference (shown above)

The paragraph explicitly contains <w:numPr> with both level and numId.

2. Via Paragraph Style

The paragraph uses a style, and the numbering level has <w:pStyle> pointing to that style:

<!-- In numbering.xml -->
<w:lvl w:ilvl="0">
    <w:pStyle w:val="Heading1"/>  <!-- Linked to Heading1 style -->
    <w:start w:val="1"/>
    <w:numFmt w:val="upperRoman"/>
    ...
</w:lvl>

<!-- In document.xml -->
<w:p>
    <w:pPr>
        <w:pStyle w:val="Heading1"/>  <!-- Just the style! -->
        <w:numPr>
            <w:numId w:val="1"/>  <!-- Level determined by style link -->
        </w:numPr>
    </w:pPr>
    ...
</w:p>
‚ö†Ô∏è Important

When numbering is linked via <w:pStyle>, a paragraph style can only be linked to ONE numbering list. This is why multiple independent lists are so tricky in Word!

‚öñÔ∏è Legal Document Examples

Traditional Legal Hierarchy

Here's a complete numbering definition for a typical legal document:

<w:abstractNum w:abstractNumId="0">
    <w:multiLevelType w:val="multilevel"/>
    
    <!-- Level 0: Article I, Article II -->
    <w:lvl w:ilvl="0">
        <w:start w:val="1"/>
        <w:numFmt w:val="upperRoman"/>
        <w:lvlText w:val="Article %1."/>
        <w:lvlJc w:val="left"/>
    </w:lvl>
    
    <!-- Level 1: Section 1.1, 1.2 -->
    <w:lvl w:ilvl="1">
        <w:start w:val="1"/>
        <w:numFmt w:val="decimal"/>
        <w:lvlText w:val="%1.%2"/>  <!-- Uses Article # too! -->
        <w:lvlJc w:val="left"/>
        <w:lvlRestart w:val="0"/>  <!-- Restart after level 0 -->
    </w:lvl>
    
    <!-- Level 2: (a), (b), (c) -->
    <w:lvl w:ilvl="2">
        <w:start w:val="1"/>
        <w:numFmt w:val="lowerLetter"/>
        <w:lvlText w:val="(%3)"/>
        <w:lvlJc w:val="left"/>
        <w:lvlRestart w:val="1"/>  <!-- Restart after level 1 -->
    </w:lvl>
    
    <!-- Level 3: (i), (ii), (iii) (romanettes) -->
    <w:lvl w:ilvl="3">
        <w:start w:val="1"/>
        <w:numFmt w:val="lowerRoman"/>
        <w:lvlText w:val="(%4)"/>
        <w:lvlJc w:val="left"/>
        <w:lvlRestart w:val="2"/>
    </w:lvl>
</w:abstractNum>

This produces:

Article I.  DEFINITIONS
    1.1  "Agreement" means...
        (a)  The parties hereto...
            (i)   First sub-item
            (ii)  Second sub-item
        (b)  Including any amendments...
    1.2  "Effective Date" means...
Article II.  REPRESENTATIONS
    2.1  The Company represents...

The lvlRestart Element

<w:lvlRestart> tells Word to restart this level's numbering after encountering a higher level:

  • w:val="0" ‚Üí Restart after Level 0
  • w:val="1" ‚Üí Restart after Level 1
üí° Legal Style Numbering

Setting <w:isLgl w:val="true"/> on a level forces "legal style" ‚Äî where higher levels show as arabic even if defined as roman. Example: "1.1.1" instead of "I.A.1"

Modern Tiered Format (1.1.1)

<w:abstractNum w:abstractNumId="1">
    <w:multiLevelType w:val="multilevel"/>
    
    <!-- Level 0: 1. -->
    <w:lvl w:ilvl="0">
        <w:start w:val="1"/>
        <w:numFmt w:val="decimal"/>
        <w:lvlText w:val="%1."/>
    </w:lvl>
    
    <!-- Level 1: 1.1 -->
    <w:lvl w:ilvl="1">
        <w:start w:val="1"/>
        <w:numFmt w:val="decimal"/>
        <w:lvlText w:val="%1.%2"/>
        <w:lvlRestart w:val="0"/>
    </w:lvl>
    
    <!-- Level 2: 1.1.1 -->
    <w:lvl w:ilvl="2">
        <w:start w:val="1"/>
        <w:numFmt w:val="decimal"/>
        <w:lvlText w:val="%1.%2.%3"/>
        <w:lvlRestart w:val="1"/>
    </w:lvl>
</w:abstractNum>

üîß Common Problems & Fixes

Problem 1: Numbering Won't Restart

‚ùå The Problem

You have two separate lists but Word continues numbering from the first list instead of restarting.

What's happening in OOXML: Both paragraph groups reference the same <w:num> instance.

The Fix: Create a second <w:num> pointing to the same <w:abstractNum>:

<!-- First list uses numId="1" -->
<w:num w:numId="1">
    <w:abstractNumId w:val="0"/>
</w:num>

<!-- Second list uses numId="2" (same abstract, new instance!) -->
<w:num w:numId="2">
    <w:abstractNumId w:val="0"/>
</w:num>

Problem 2: Font Corruption in Numbered Lists

‚ùå The Problem

Numbers suddenly appear in a different font (often Symbol or Wingdings).

What's happening in OOXML: The <w:rPr> inside a level definition has corrupted font settings:

<w:lvl w:ilvl="0">
    ...
    <w:rPr>
        <!-- This is the corruption! -->
        <w:rFonts w:ascii="Symbol" w:hAnsi="Symbol"/>
    </w:rPr>
</w:lvl>

The Fix: Remove or correct the <w:rFonts> element in the level's run properties.

Problem 3: Styles Linked to Wrong Numbering

‚ùå The Problem

Applying "Heading 1" gives incorrect numbering or no numbering at all.

What's happening: The <w:pStyle> reference in numbering.xml doesn't match styles.xml.

The Fix: Ensure the style name in <w:pStyle w:val="Heading1"/> exactly matches the <w:name w:val="Heading 1"/> in styles.xml (note: spaces matter!).

Problem 4: Too Many Numbering Definitions

‚ö†Ô∏è The Problem

Document becomes slow or unstable; numbering behaves unpredictably.

What's happening: Word accumulates orphaned <w:abstractNum> and <w:num> elements over time, especially from copy/paste operations.

The Fix: Clean up numbering.xml:

  1. Identify which numId values are actually used in document.xml
  2. Remove unused <w:num> elements
  3. Remove <w:abstractNum> elements not referenced by any remaining <w:num>
üí° Python Cleanup Script

Using python-docx and lxml, you can programmatically clean orphaned numbering definitions. Unzip the .docx, parse numbering.xml, cross-reference with document.xml, remove orphans, rezip.

üêç Bonus: Python Example

Here's how to read numbering definitions using Python:

from zipfile import ZipFile
from lxml import etree

# OOXML namespace
NSMAP = {'w': 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'}

def read_numbering(docx_path):
    with ZipFile(docx_path, 'r') as z:
        # Read numbering.xml
        numbering_xml = z.read('word/numbering.xml')
        root = etree.fromstring(numbering_xml)
        
        # Find all abstract numbering definitions
        abstract_nums = root.findall('.//w:abstractNum', NSMAP)
        
        for abstract in abstract_nums:
            abs_id = abstract.get('{http://schemas.openxmlformats.org/wordprocessingml/2006/main}abstractNumId')
            print(f"Abstract Num ID: {abs_id}")
            
            # Get levels
            levels = abstract.findall('w:lvl', NSMAP)
            for lvl in levels:
                ilvl = lvl.get('{http://schemas.openxmlformats.org/wordprocessingml/2006/main}ilvl')
                num_fmt = lvl.find('w:numFmt', NSMAP)
                fmt = num_fmt.get('{http://schemas.openxmlformats.org/wordprocessingml/2006/main}val') if num_fmt is not None else 'none'
                print(f"  Level {ilvl}: {fmt}")

read_numbering('contract.docx')

Output:

Abstract Num ID: 0
  Level 0: upperRoman
  Level 1: decimal
  Level 2: lowerLetter
  Level 3: lowerRoman
