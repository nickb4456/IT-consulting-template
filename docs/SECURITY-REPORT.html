2. Quote Injection in Event Handlers Fixed

The delete button used inline onclick handlers with IDs that weren't properly escaped for quote characters, allowing JavaScript injection via localStorage manipulation.

// Vulnerable: escapeHtml didn't escape quotes
onclick="deleteUserScheme('${escapeHtml(id)}')"

// Attacker sets localStorage ID to:
'); alert(document.cookie); ('

// Rendered as:

‚úì Fix Applied

escapeHtml() now escapes single quotes (' ‚Üí &#39;) and double quotes (" ‚Üí &quot;).

3. Missing Content Security Policy Fixed

Without CSP, any XSS vulnerability allows attackers to load external scripts, send data to any domain, and bypass same-origin restrictions.

<!-- CSP now restricts resource loading -->
<meta http-equiv="Content-Security-Policy" content="
  default-src 'self';
  script-src 'self' 'unsafe-inline' https://appsforoffice.microsoft.com;
  connect-src 'self' https://*.execute-api.us-east-1.amazonaws.com;
">

‚úì Fix Applied

CSP header added. Even if XSS exists, attackers cannot load external scripts or exfiltrate data to unauthorized domains.

4. Office.js Without Subresource Integrity Cannot Fix

The Office.js library is loaded from Microsoft's CDN without an integrity hash. If Microsoft's CDN is compromised, malicious code could be injected into all Office add-ins.

<!-- Current (no integrity check) -->
<script src="https://appsforoffice.microsoft.com/lib/1/hosted/office.js"></script>

<!-- Ideal (but Microsoft doesn't provide hash) -->
<script src="..." integrity="sha384-..."></script>

Data stored in localStorage can be modified by anyone with browser access. Malicious data could be injected to exploit parsing vulnerabilities.
