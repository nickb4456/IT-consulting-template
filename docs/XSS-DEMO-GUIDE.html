üî¨ How It Works

1

Hidden Payload

Attacker embeds JavaScript in a style name within the Word document's XML structure. The document looks completely normal.

2

Style Name Displayed

When software shows the numbering picker or style list, it renders the style name as HTML without escaping special characters.

3

Code Executes

The browser interprets the injected HTML/JavaScript. The attacker's code runs with full access to the add-in's capabilities.

4

Document Modified

Using the Office JavaScript API, the attacker silently modifies the document ‚Äî changing text, numbers, dates, or clauses.

// VULNERABLE: Renders style name as HTML element.innerHTML = `Style: ${styleName}`; // SECURE: Escapes special characters first function escapeHtml(text) { return String(text) .replace(/&/g, '&amp;') .replace(/'&lt;') .replace(/>/g, '&gt;') .replace(/"/g, '&quot;') .replace(/'/g, '&#39;'); } element.innerHTML = `Style: ${escapeHtml(styleName)}`;

Additionally, a Content Security Policy header prevents inline script execution even if escaping is somehow bypassed.
